# Автоматический деплой проекта

## Варианты автоматизации деплоя

### 1. GitHub Pages (Рекомендуется для начала)

**Преимущества:**
- ✅ Бесплатно
- ✅ Автоматический деплой при каждом push в ветку `main`
- ✅ HTTPS по умолчанию
- ✅ Простая настройка

**Что нужно:**
1. В настройках репозитория GitHub:
   - Settings → Pages
   - Source: Deploy from a branch
   - Branch: `main` / `/ (root)`
   - Save

2. Сайт будет доступен по адресу:
   `https://moskvolod3-beep.github.io/Mediavelichie/`

**Настройка:**
- Автоматически работает после включения в настройках GitHub
- Каждый `git push` автоматически обновляет сайт

---

### 2. Netlify (Альтернатива)

**Преимущества:**
- ✅ Бесплатный план
- ✅ Автоматический деплой при push
- ✅ Превью для pull requests
- ✅ Кастомный домен
- ✅ CDN по умолчанию

**Что нужно:**
1. Зарегистрироваться на [netlify.com](https://netlify.com)
2. Подключить GitHub репозиторий
3. Настройки деплоя:
   - Build command: (оставить пустым, т.к. проект статический)
   - Publish directory: `/` (корень проекта)
4. Deploy!

**Настройка:**
- Автоматически деплоит при каждом push в `main`
- Превью для каждой ветки

---

### 3. Vercel (Альтернатива)

**Преимущества:**
- ✅ Бесплатный план
- ✅ Автоматический деплой
- ✅ Отличная производительность

**Что нужно:**
1. Зарегистрироваться на [vercel.com](https://vercel.com)
2. Импортировать GitHub репозиторий
3. Настройки:
   - Framework Preset: Other
   - Root Directory: `./`
   - Build Command: (пусто)
   - Output Directory: `./`
4. Deploy!

---

## Рекомендация

Для начала используйте **GitHub Pages** — это самый простой вариант:
1. Включите в настройках репозитория
2. Готово! Сайт будет автоматически обновляться при каждом `git push`

## Настройка кастомного домена (опционально)

Если у вас есть домен (например, `mediavelichie.ru`):

### Для GitHub Pages:
1. В настройках Pages добавьте кастомный домен
2. Настройте DNS записи у регистратора домена:
   - A записи: `185.199.108.153`, `185.199.109.153`, `185.199.110.153`, `185.199.111.153`
   - Или CNAME: `moskvolod3-beep.github.io`

### Для Netlify/Vercel:
- Добавьте домен в настройках проекта
- Следуйте инструкциям по настройке DNS

## GitHub Actions (для продвинутых)

Если нужны дополнительные действия при деплое (например, минификация, проверки), можно использовать GitHub Actions.

Создайте файл `.github/workflows/deploy.yml`:

```yaml
name: Deploy

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
```

Но для статического сайта это обычно не требуется.
